passwd: system password manager for PyOS

SYNOPSIS
    passwd <new_password>

DESCRIPTION
    passwd is a system utility used within the PyOS shell to change the system password.
    The current password is stored in `imp/system.json` and verified before allowing changes.

    When invoked from the shell, passwd is dispatched as a registered sbin command through
    the command dispatcher in `shell.py` based on membership in `sbin/register.json`.

USAGE
    passwd <new_password>
        Changes the system password to <new_password>.

        If the current password is the default ("root"), the new password is set immediately
        without verification.

        If a custom password has already been set, you will be prompted for the current
        password before the change is allowed.

EXAMPLES
    First-time password change (default "root" active):
        system@host$ passwd mysecurepassword

    Password change after custom password is set:
        system@host$ passwd mynewpassword
        Current password: (enter current password, then new password is set)

ERROR HANDLING
    - If no new password is provided, an error is printed.
    - If current password verification fails, the change is rejected.
    - If system.json cannot be read or written, operation fails with error.

FILES
    imp/system.json
        System configuration file containing the encrypted or stored password.

NOTES
    - The default password is "root" for initial system setup.
    - It is strongly recommended to change the password on first shell startup.
    - Password changes are persisted immediately to imp/system.json.
